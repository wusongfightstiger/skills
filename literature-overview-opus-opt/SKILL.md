# 天文学术文献概览生成（优化版）

生成天体物理及天文技术与方法领域学术论文的中文文献概览，输出为 Word 文档。

## 第一步：获取 PDF 路径

向用户询问要分析的 PDF 文献的完整路径（含文件名）。如果用户在调用命令时已经提供了路径，直接使用。

## 第二步：读取与理解 PDF

使用 Read 工具逐页读取 PDF 文件，仔细查看每一页。天文论文中的图表和公式往往承载最关键的信息（光变曲线、光谱图、HR 图、拟合结果等），不能跳过。

阅读时关注并提取：

- 论文标题（通常在首页顶部）
- 作者列表与单位
- 发表信息：期刊名、卷号、页码、年份、DOI
- 摘要（Abstract）
- 正文各章节的核心内容，同时记录每个章节对应的编号（如 Section 2、Section 3.1 等）
- 图片与图注（Figure + Caption）
- 表格与表题（Table + Title）
- 关键公式
- 参考文献列表（References）

## 第三步：信息提取与翻译

### 3.1 标题与摘要翻译

如果论文是英文的：
- 将标题翻译为通顺的中文，同时保留英文原文
- 将摘要全文翻译为中文，力求准确达意
- 专业术语采用"中文译名（English original）"格式，例如："恒星形成率（Star Formation Rate, SFR）"

如果论文本身是中文的，标题和摘要保持原样。

### 3.2 结构化信息提取

从论文中提取以下维度的信息。所有内容必须忠实于原文——逐条核查，确保每个论点都能在原文中找到对应来源，不做引申、不发挥、不杜撰。

每个维度的标题后必须标注该内容在原文中对应的章节编号，格式为 `(Section X)` 或 `(Section X.Y)`。如果某个维度的内容分散在多个章节中，列出所有相关章节号，如 `(Section 2, 4.1)`。

1. **基本信息**
   - 作者：列出全部作者，英文原名 + 中文译名（如有把握）
   - 发表时间：精确到年份（如有月份也标注）
   - 发表刊物：期刊全称 + 缩写
   - DOI

2. **研究背景**
   - 该研究所处领域的发展现状
   - 前人工作的主要成果和局限
   - 尚未解决的关键问题

3. **研究意义**
   - 理论层面的贡献
   - 观测/实验/应用层面的价值

4. **研究目标**
   - 研究要回答的核心科学问题
   - 具体的研究目标和预期成果

5. **研究所用数据**
   - 数据来源（巡天项目、数据库、自有观测、模拟数据等）
   - 数据参数（波段、分辨率、时间跨度、样本量、信噪比等关键参数）
   - 数据处理（是否经过 reduction pipeline、校准、筛选等预处理，具体方法是什么）

6. **研究方法** ← 重点，需要详细展开
   - 分析方法、物理模型、数值模拟等
   - 观测设备/望远镜/探测器（如适用）
   - 关键参数的选取和依据
   - 实验设计或观测策略
   - 如果涉及重要公式，记录公式内容及各符号含义

7. **全部结论** ← 重点，需要详细展开
   - 逐条列出论文中的所有结论，不遗漏
   - 关键定量结果（数值、置信区间、统计显著性等）
   - 与前人工作的比较和一致性/差异性

8. **创新点** ← 重点，需要详细展开
   - 方法创新：新算法、新技术、新观测策略
   - 理论创新：新模型、新解释、新预测
   - 应用创新：已有方法的新应用场景

9. **研究展望**
   - 作者提出的后续研究方向
   - 现有工作的局限性
   - 尚未解决的问题

### 3.3 引用文献处理

在撰写文献概览时，如果引用了原文中带有引用标记的内容（如 "Smith et al. 2020 发现..."），在概览中保留这个引用标记，并将对应的完整文献信息收录在最后的参考文献部分。

只收录概览中实际引用到的文献，不照搬原文的全部参考文献列表。

### 3.4 ADS 引文格式

为被概览的论文本身生成一条标准 ADS 引文，格式为：

```
作者1, 作者2, ... 年份, 标题, 期刊缩写, 卷号, 页码/文章编号. doi:DOI号
```

示例：
```
Wang, L., Zhang, Y., & Li, H. 2023, ApJ, 945, 112. doi:10.3847/1538-4357/ac1234
```

多于三位作者时也全部列出，不使用 "et al." 缩写。

## 第四步：生成 Word 文档

使用 docx 技能生成结构完整的 Word 文档。文档结构如下：

```
标题：文献概览：[论文中文标题]

一、基本信息（表格形式）
┌──────────────┬─────────────────────────────┐
│ 项目         │ 内容                         │
├──────────────┼─────────────────────────────┤
│ 标题（中文） │ [中文标题]                    │
│ 标题（英文） │ [英文标题]                    │
│ 作者         │ [作者列表]                    │
│ 发表时间     │ [年份]                        │
│ 发表刊物     │ [期刊名称]                    │
│ DOI          │ [DOI]                         │
│ ADS 引文     │ [标准ADS格式引文]             │
└──────────────┴─────────────────────────────┘

二、摘要
[中文翻译摘要]
（附：英文原文摘要）

三、研究背景 (Section X)
[内容]

四、研究意义 (Section X)
[内容]

五、研究目标 (Section X)
[内容]

六、研究所用数据 (Section X)
[数据来源、数据参数、数据处理流程]

七、研究方法 (Section X) ★
[详细内容，含图表和公式]

八、全部结论 (Section X) ★
[逐条列出所有结论，含关键数据]

九、创新点 (Section X) ★
[详细内容]

十、研究展望 (Section X)
[内容]

十一、参考文献
[概览中引用到的文献，按ADS格式排列]
```

标记 ★ 的章节是重点，内容应当最为详尽。

## 第五步：嵌入图片、表格与公式

天文论文的核心结果往往通过图表呈现（光变曲线、光谱、成像、拟合图等），这一步至关重要。

### 图片处理
- 从原 PDF 中提取关键图片，嵌入到 Word 文档中与其讨论内容相邻的位置
- 保留原图注（Caption），如果是英文则翻译为中文并附原文
- 每个图旁标注原文编号（如"图3 / Figure 3"）

### 表格处理
- 将原文中的关键数据表格重建为 Word 表格，放在对应讨论内容附近
- 保留表题，如果是英文则翻译为中文并附原文
- 每个表旁标注原文编号（如"表2 / Table 2"）

**大型数据表排除规则**：论文中的源表（source table）、星表（catalog）等大型数据表格不提取到概览中。对于这类表格，在文档中注明表格名称，并给出论文中作者提供的数据发布链接（如 VizieR、CDS、Zenodo 等）。如果原文未提供数据链接，注明"原文未提供数据发布链接"。

### 公式处理
- 对于重要公式，优先作为图片嵌入
- 如无法提取图片，用文字准确描述公式内容及各符号含义

所有图片、表格、公式直接嵌入 Word 文档正文中对应文字的附近，不集中放在文档末尾或单独文件夹中。

## 第六步：保存与交付

将文档保存为 `文献概览_[原文件名].docx`，其中 `[原文件名]` 是输入 PDF 文件的名称（不含 .pdf 扩展名）。

保存到与输入 PDF 相同的目录下。告知用户文件的完整保存路径。

## 质量检查清单

生成文档前，对照以下清单自检：

- [ ] 标题和摘要的中文翻译是否准确通顺？
- [ ] 所有提取信息是否忠实于原文，无杜撰？
- [ ] 每个内容维度标题后是否标注了原文章节号 (Section X)？
- [ ] 研究方法、全部结论、创新点三个重点章节是否足够详细？
- [ ] "研究所用数据"是否独立成节，包含数据来源、参数、处理流程？
- [ ] 图片、表格是否已嵌入到文档中对应位置？
- [ ] 大型数据表（源表、星表）是否已排除，并给出原文数据链接？
- [ ] 概览中引用的文献是否都收录在参考文献部分？
- [ ] ADS 引文格式是否正确？
- [ ] 专业术语翻译是否附有英文原文？
- [ ] 文件名格式是否为 `文献概览_原文件名.docx`？
