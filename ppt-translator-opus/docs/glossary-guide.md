# 术语表调用规则

## 文件位置

默认术语表：`~/.claude/skills/ppt-translator-opus/glossary.csv`

## 调用优先级

| 优先级 | 参数 | 行为 |
|--------|------|------|
| 1 | `--no-glossary` | 完全禁用术语表，不加载任何术语 |
| 2 | `--glossary <path>` | 使用用户指定的术语表，忽略默认术语表 |
| 3 | 无参数 | 自动加载默认术语表 `glossary.csv`（如果文件存在） |

如果默认术语表文件不存在且未指定 `--glossary`，则不使用术语表（等同于 `--no-glossary`）。

## 术语表格式

### CSV 格式（推荐）

文件扩展名：`.csv`

```csv
中文,英文,领域
电路,circuit,电气工程
基尔霍夫定律,Kirchhoff's law,电气工程
赫罗图,Hertzsprung-Russell diagram,天文学
```

字段说明：
- `中文`：必填，中文术语
- `英文`：必填，对应的英文翻译
- `领域`：可选，术语所属领域（用于未来按领域筛选）

### TXT 格式

文件扩展名：`.txt`

```
电路	circuit
基尔霍夫定律	Kirchhoff's law
赫罗图	Hertzsprung-Russell diagram
```

- 每行一个术语对
- 中英文之间用 Tab 分隔（也支持空格分隔）
- `#` 开头的行视为注释，自动跳过
- 无领域字段

## 术语匹配机制

翻译时不做预替换。而是在每张幻灯片翻译前：

1. 提取当前幻灯片的全部文本
2. 扫描文本中出现了哪些术语表中的中文词条
3. 仅将命中的术语作为"术语参考"传给 LLM
4. LLM 在翻译时参考这些术语，但由 LLM 自行决定最终表述

这种方式的优点：
- 不破坏原文语句结构（预替换可能导致断句问题）
- LLM 可以根据上下文灵活调整（如"电路分析"整体翻译为 "circuit analysis" 而非拆开）
- 只传递相关术语，不浪费 token

## 手动维护

直接编辑 `glossary.csv` 即可：

**添加术语：** 在文件末尾新增一行
```csv
新术语,new term,领域
```

**删除术语：** 删除对应行

**修改术语：** 直接修改对应行的英文翻译

注意事项：
- 确保 CSV 编码为 UTF-8
- 第一行必须是表头 `中文,英文,领域`
- 术语中如果包含逗号，需要用双引号包裹：`"含,逗号的术语",translation,领域`
