# PPT翻译工具设计文档

**日期**: 2026-03-21
**项目**: 中文PPT翻译成英文PPT

---

## 1. 目标概述

创建一个工具，将中文电气工程课程PPT批量翻译为英文版本，翻译质量达到学术/专业级别。

---

## 2. 核心需求

| 需求项 | 选择 |
|--------|------|
| 翻译质量 | 学术/专业级（MiniMax API） |
| 工作流程 | 批量处理 |
| 翻译后端 | MiniMax API |
| 术语表管理 | 电路术语表.csv，用户确认后扩展，新术语需人工确认 |
| 处理粒度 | 文本框级（每文本框独立翻译） |
| 批量处理 | 单PPT批量（一个PPT的所有幻灯片/文本框批量翻译） |
| PPT结构 | 完全保留（图片、形状、颜色、布局、动画） |
| 字体处理 | 全英文，统一西文字体（Arial/Calibri） |
| 字符扩张 | 启用autoFit，设置字号缩小限制 |
| 公式处理 | 纯符号保持原样；公式名称（如欧拉定律）整体翻译 |
| 编号处理 | 保持原编号系统不变 |
| 人名地名 | 使用拼音 |
| 错误处理 | 文本框级独立回滚（单个文本框失败则该框回滚，不影响其他文本框） |
| 输出路径 | 源文件目录/Translated/第X章_英文版.pptx |

## 5. 模块说明

| 模块 | 职责 |
|------|------|
| `cli.py` | 命令行入口，参数解析 |

## 6. 核心流程

```
1. 读取中文PPT
       ↓
2. Unpack: 解压为XML文件
       ↓
3. 解析所有文本框，提取可翻译文本
       ↓
4. 对每个文本框：
   a. 检查术语表，预替换已知术语（如"电阻"→"resistor "，保留空格）
   b. 调用MiniMax API翻译
   c. 记录新发现的术语（不中断翻译）
       ↓
5. Pack: 重新打包为PPTX
       ↓
6. 输出到 源文件目录/Translated/第X章_英文版.pptx
       ↓
7. 汇总新术语，等待用户确认
       ↓
8. 用户确认后 → 添加到术语表
```

## 7. 术语表管理

**存储**: `电路术语表.csv`

**字段**:
- 中文术语
- 英文翻译
- 领域
- 添加日期
- 是否已确认

**新术语流程**:
```
1. 完成整个PPT翻译（不中断）
2. 翻译结束后，汇总发现的新术语
3. 询问用户：
   "发现N个新术语，是否添加到术语表？"
     ↓
   用户确认 → 所有新术语添加到术语表（状态=已确认）
   用户拒绝 → 不添加
   用户部分确认 → 选择性添加
     ↓
4. 术语表更新完成
```

## 8. PPT处理细节

### 8.1 unpack/edit/pack

```
Unpack:  input.pptx → temp_dir/
         (包含 ppt/slides/slide*.xml 等)

Edit:    遍历XML，定位文本节点，替换翻译内容
         处理字体、autoFit属性

Pack:    zip temp_dir/ → output.pptx
```

### 8.2 字体处理

| 内容类型 | 字体处理 |
|----------|----------|
| 翻译后英文文本 | 西文字体（Arial/Calibri） |
| 图片中的文字 | 不处理（无法提取） |

### 8.3 字符扩张处理

- 启用 `autoFit: text` 属性
- 设置字号缩小上限（如最多缩小50%）
- 设置最小字号下限（如8pt）

### 8.4 特殊内容

- **公式**: 纯符号（E=mc²）保持不变；中文公式名称（欧拉定律）翻译
- **编号**: 1-1, (a), (b), ① ② ③ 保持原样
- **人名地名**: 翻译为拼音

---

## 9. 错误处理策略

- **单个文本框API调用失败** → 该文本框内容回滚至原文，标记失败项，继续处理其他文本框
- **文件写入失败** → 回滚，保留原文件
- **XML解析错误** → 回滚，报告错误位置
- **术语确认** → 英文PPT生成完成后进行，用户确认后更新术语表
- **批量处理单位** → 单个PPT文件
