# 天文学术文献概览生成

生成天体物理及天文技术与方法领域学术论文的中文文献概览，输出为 Word 文档。

## 第一步：获取 PDF 路径

向用户询问要分析的 PDF 文献的完整路径（含文件名）。如果用户在调用命令时已经提供了路径，直接使用即可。

## 第二步：读取与理解 PDF

使用 Read 工具读取 PDF 文件。PDF 读取工具能够处理文本和视觉内容，逐页提取信息。

阅读时需要关注并提取的核心内容：

- **论文标题**（通常在首页顶部）
- **作者列表与单位**
- **发表信息**：期刊名、卷号、页码、年份、DOI
- **摘要**（Abstract）
- **正文各章节**的核心内容
- **图片与图注**（Figure + Caption）
- **表格与表题**（Table + Title）
- **关键公式**
- **参考文献列表**（References）

阅读 PDF 时，对每一页都要仔细查看。天文论文中的图表和公式往往承载着最关键的信息（如光变曲线、光谱图、HR图、拟合结果等），不能跳过。

## 第三步：信息提取与翻译

### 3.1 标题与摘要翻译

如果论文是英文的：
- 将标题翻译为通顺的中文，同时保留英文原文
- 将摘要全文翻译为中文，力求准确达意
- 对于专业术语的翻译，如果拿不准，采用"中文译名（English original）"的格式，例如："恒星形成率（Star Formation Rate, SFR）"

如果论文本身是中文的，标题和摘要保持原样。

### 3.2 结构化信息提取

从论文中提取以下八个维度的信息。所有内容必须忠实于原文——逐条核查，确保每个论点都能在原文中找到对应来源，不做引申、不发挥、不杜撰。

1. **基本信息**
   - 作者：列出全部作者，英文原名 + 中文译名（如有把握）
   - 发表时间：精确到年份（如有月份也标注）
   - 发表刊物：期刊全称 + 缩写
   - DOI

2. **研究背景**
   - 该研究所处领域的发展现状
   - 前人工作的主要成果和局限
   - 尚未解决的关键问题

3. **研究意义**
   - 理论层面的贡献
   - 观测/实验/应用层面的价值

4. **研究目标**
   - 研究要回答的核心科学问题
   - 具体的研究目标和预期成果

5. **研究方法** ← 重点，需要详细展开
   - 观测设备/望远镜/探测器（如适用）
   - 数据来源（巡天项目、数据库、自有观测等）
   - 数据处理流程（reduction pipeline、校准方法等）
   - 分析方法、物理模型、数值模拟等
   - 关键参数的选取和依据
   - 如果涉及重要公式，记录公式内容

6. **主要结论** ← 重点，需要详细展开
   - 核心发现，逐条列出
   - 关键定量结果（数值、置信区间、统计显著性等）
   - 与前人工作的比较和一致性/差异性

7. **创新点** ← 重点，需要详细展开
   - 方法创新：新算法、新技术、新观测策略
   - 理论创新：新模型、新解释、新预测
   - 应用创新：已有方法的新应用场景

8. **研究展望**
   - 作者提出的后续研究方向
   - 现有工作的局限性
   - 未来可改进之处

### 3.3 引用文献处理

在撰写文献概览时，如果引用了原文中某处带有引用标记的内容（如 "Smith et al. 2020 发现..."），在概览中也要保留这个引用标记，并将对应的完整文献信息收录在最后的参考文献部分。

只需收录概览中实际引用到的文献，不需要照搬原文的全部参考文献列表。

### 3.4 ADS 引文格式

为被概览的论文本身生成一条标准 ADS（Astrophysics Data System，参考 https://ui.adsabs.harvard.edu ）引文，格式为：

```
作者1, 作者2, ... 年份, 标题, 期刊缩写, 卷号, 页码/文章编号. doi:DOI号
```

**示例**：
```
Wang, L., Zhang, Y., & Li, H. 2023, ApJ, 945, 112. doi:10.3847/1538-4357/ac1234
```

多于三位作者时也全部列出，不使用 "et al." 缩写（这里是完整引文，不是正文中的引用）。

## 第四步：生成 Word 文档

使用 docx 技能生成结构完整的 Word 文档。文档结构如下：

```
标题：文献概览：[论文中文标题]

一、基本信息（表格形式）
┌──────────────┬─────────────────────────────┐
│ 项目         │ 内容                         │
├──────────────┼─────────────────────────────┤
│ 标题（中文） │ [中文标题]                    │
│ 标题（英文） │ [英文标题]                    │
│ 作者         │ [作者列表]                    │
│ 发表时间     │ [年份]                        │
│ 发表刊物     │ [期刊名称]                    │
│ DOI          │ [DOI]                         │
│ ADS 引文     │ [标准ADS格式引文]             │
└──────────────┴─────────────────────────────┘

二、摘要
[中文翻译摘要]
（附：英文原文摘要）

三、研究背景
[内容]

四、研究意义
[内容]

五、研究目标
[内容]

六、研究方法 ★
[详细内容，含图表和公式]

七、主要结论 ★
[详细内容，含关键数据]

八、创新点 ★
[详细内容]

九、研究展望
[内容]

十、参考文献
[概览中引用到的文献，按ADS格式排列]
```

标记 ★ 的章节是重点，内容应当最为详尽。

## 第五步：嵌入图片、表格与公式

这一步至关重要——天文论文的核心结果往往通过图表呈现（光变曲线、光谱、成像、拟合图等）。

处理原则：
- **图片**：从原 PDF 中提取关键图片，嵌入到 Word 文档中与其讨论内容相邻的位置。保留原图注（Caption），如果是英文则翻译为中文并附原文。
- **表格**：将原文中的关键数据表格重建为 Word 表格，放在对应讨论内容附近。保留表题。
- **公式**：对于文中出现的重要公式，优先作为图片嵌入；如无法提取图片，则用文字准确描述公式内容及各符号含义。

关键要求：所有图片、表格、公式直接嵌入 Word 文档正文中对应文字的附近，而不是放在单独的文件夹或集中放在文档末尾。每个图表旁应有简短说明，指出它展示了什么信息以及在原文中的编号（如"图3"、"表2"）。

## 第六步：保存与交付

将文档保存为 `文献概览_[原文件名].docx`，其中 `[原文件名]` 是输入 PDF 文件的名称（不含 .pdf 扩展名）。

保存到与输入 PDF 相同的目录下。

告知用户文件的完整保存路径。

## 质量检查清单

生成文档前，对照以下清单自检：

- [ ] 标题和摘要的中文翻译是否准确通顺？
- [ ] 所有提取信息是否忠实于原文，无杜撰？
- [ ] 研究方法、主要结论、创新点三个重点章节是否足够详细？
- [ ] 图片、表格是否已嵌入到文档中对应位置（而非单独文件夹）？
- [ ] 概览中引用的文献是否都收录在参考文献部分？
- [ ] ADS 引文格式是否正确？
- [ ] 专业术语翻译是否附有英文原文？
- [ ] 文件名格式是否正确？
